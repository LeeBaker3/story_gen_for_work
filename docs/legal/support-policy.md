# Support / Contact Policy

Effective date: 2026-05-13

This policy describes how support requests are handled for the current repository baseline.

## Support Channels

Use the repository issue tracker for:
- bug reports;
- documentation gaps;
- feature requests that do not require private account details.

Use a private reporting path for security-sensitive issues when one is published by the maintainers.
Do not post secrets, passwords, tokens, or full private data in public issues.

## What Support Covers

Support can help with:
- login and basic account access questions;
- story generation, editing, and export issues;
- character and upload workflow questions;
- moderation or content removal questions;
- documentation and setup questions for the current codebase.

## What Support Does Not Promise

This first-pass policy does not include a formal SLA or guaranteed response time.
Responses are best effort and depend on maintainer availability.
The policy also does not promise assistance with third-party outages, local environment problems, or custom deployment setups.

## Escalation

If a request involves account safety, suspected abuse, or a rights complaint, include only the minimum necessary details and request private follow-up.
